Catalyst:
a WASM GC Open Smalltalk virtual machine with LLM-assisted method translation

This work and presentation are at an early stage. Please jump in at any time with questions and comments. Follow new developments at my blog... thiscontext.com.

How can Smalltalk participate in the WebAssembly module ecosystem? In a web browser, how much faster can it be than a JavaScript implementation?


Catalyst is enabled by the Epigram compiler framework. The Smalltalk support reifies grammar tokens, code generation units, instruction sets, and instructions, to provide compilation, decompilation to multiple languages, instruction printing, and interpretation.


WASM GC is strictly and statically typed. Through the use of pragmas, we can annotate classes and method as to the types of their fields and variables.


SmallIntegers are of WASM GC reference type i31. Every other object is of a user-defined struct type; the supertype of all those types is WASM GC reference type (ref eq), for entities that can be compared. This type is used when passing Smalltalk objects between WASM modules.


Catalyst has a test harness, an interactive web interface for benchmarking and debugging. It provides real-time feedback on method translations and optimization impacts.


Catalyst's method translation uses LLMs to infer optimal bytecode sequences before translating them to WASM GC. The current benchmark method which is sufficiently expensive for time profiling, and amenable to inference. It only uses a few instructions in the instruction set.


I'm currently completing the Smalltalk simulation's support for the "V3 plus closures" instruction set, and ensuring that it decompiles to a working WASM GC implementation.
