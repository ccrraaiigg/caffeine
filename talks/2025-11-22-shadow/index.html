Shadow:
a Manifold for Collaborative Agentic Livecoding

This work and presentation are at an early stage. Please jump in at any time with questions and comments. Follow new developments at my blog... thiscontext.com.

How could an AI agent collaborate with a human livecoder?


Shadow is a manifold of contextual cues to augment the collaborative livecoding skills of an AI agent. It has an MCP server with local tools and resources, steering documents, and a scheme for externalizing the agent's context window.


the Model Context Protocol: access to local tools and resources (and prompts)


structured steering documents


The steering documents establish a scheme for externalizing the agent's context window as often as possible, to increase accuracy and human accessibility. This is context management; several other projects have interesting approaches to this (e.g., AWS Kiro, Google Antigravity, letta.com)


The agent's context window is externalized into a simple file hierarchy, each file using an agent-friendly format appropriate to its purpose. For example, additional steering documents use Markdown format, and a ledger dictionary binding Smalltalk variable names to remote object IDs uses JSON.


There are only two MCP tools: one that returns the ID of the first exposed object in a live Squeak system, and another that sends a message to a given exposed object. As the agent builds up its knowledge of the system, it becomes a useful collaborator.
